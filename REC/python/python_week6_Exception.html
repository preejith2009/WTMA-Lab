Week 5 - Exception Handling in Python

Python Exception Handling allows a program to handle unexpected errors during execution without terminating abruptly. Instead of stopping the program completely, Python provides mechanisms to detect errors, handle them properly, and continue execution whenever possible.

Exception handling improves program reliability, prevents crashes, and helps developers write robust applications.

Difference Between Errors and Exceptions

Errors and exceptions are both problems that occur in a program, but they differ in severity and handling. 

Error                           Exception

Serious problem                 Runtime problem

Cannot usually be handled       Can be handled

Causes program termination      Program can continue

Example:
Syntax Error                    ZeroDivisionError
Indentation Error               ValueError
  • Error: Errors are serious problems in a program that generally cannot be handled during execution. Examples include syntax errors and indentation errors.
  • Exception: Exceptions are runtime problems that occur during program execution and can be handled using exception handling techniques.

Types of Errors in Python

1. Syntax Errors

Syntax errors occur when Python rules are violated while writing the program. The interpreter detects these errors before execution.

Example
print("Hello"
Output : SyntaxError

2. Runtime Errors

Runtime errors occur during program execution. These errors generate exceptions.

Example
a = 10
b = 0
print(a / b)
Output : ZeroDivisionError

3. Logical Errors

Logical errors occur when the program executes successfully but produces incorrect output. These errors are difficult to identify because no error message is generated.

Example
a = 10
b = 20

print("Addition =", a - b)
Output : Addition = -10

What is Exception Handling?

Exception handling is a mechanism used to handle runtime errors in a program. Python provides special keywords such as try, except, else, and finally to manage exceptions effectively.

When an exception occurs, Python transfers control to the appropriate exception handling block instead of terminating the program abruptly.

Advantages of Exception Handling

  • Prevents abnormal termination of programs
  • Improves program reliability and stability
  • Helps in debugging and identifying errors
  • Allows graceful handling of unexpected situations
  • Improves user experience
  • Makes programs more secure and robust

Python Exception Handling Keywords

try       -> Used to test code for errors

except    -> Used to handle the error

else      -> Executes if no exception occurs

finally   -> Executes always

raise      -> Used to generate exceptions manually

The try and except Blocks

The try block contains code that may generate an exception. If an exception occurs, the control moves to the except block.

Syntax
try:
    # risky code

except:
    # handling code
Example
try:
    a = 10
    b = 0
    print(a / b)

except:
    print("Cannot divide by zero")
Output : Cannot divide by zero 
except Exception as e:
    print("Error:", e)

Handling Multiple Exceptions

Python allows handling multiple exceptions using multiple except blocks.

Example
try:
    num = int(input("Enter Number: "))
    print(10 / num)

except ZeroDivisionError:
    print("Division by zero is not allowed")

except ValueError:
    print("Invalid Input")

Using else Block

The else block executes only when no exception occurs in the try block.

Example
try:
    a = 10
    b = 2
    print(a / b)

except:
    print("Error")

else:
    print("Division Successful")
Output : 5.0 Division Successful

Using finally Block

The finally block always executes whether an exception occurs or not. It is mainly used for cleanup activities such as closing files and releasing resources.

Example
try:
    print(10 / 2)

except:
    print("Error")

finally:
    print("Program Ended")
Output : 5.0 Program Ended

Raising Exceptions using raise

The raise keyword is used to generate exceptions manually.

Example
age = -5

if age < 0:
    raise ValueError("Age cannot be negative")

Using pass Statement in Exception Handling

The pass statement is used when the programmer does not want to execute any code for a particular exception. It acts as a placeholder and prevents the program from showing an error.

The pass statement is useful during program development when exception handling logic is not yet implemented.

Syntax
try:
    # risky code

except:
    pass
Program
try:

    num1 = 10
    num2 = 0

    result = num1 / num2

    print(result)

except ZeroDivisionError:

    pass

print("Program Continues")
Logic
  • The program attempts division by zero.
  • ZeroDivisionError occurs during execution.
  • The except block contains only pass statement.
  • No error message is displayed.
  • The program execution continues normally.
Explanation
  • pass is a null statement in Python.
  • It performs no action when executed.
  • The exception is ignored silently.
  • The program continues execution after the exception block.
Output : Program Continues

Program 1 – Valid Positive Integer Average Calculator

Write a program that calculates the average of integers entered by the user. The program should validate inputs using exception handling.

Program
try:
    n = int(input())

    if n <= 0:
        print("Error: Length must be positive")
        exit()

except ValueError:
    print("Error: Enter numeric value")
    exit()

numbers = []

for i in range(n):

    while True:

        try:
            num = int(input())
            numbers.append(num)
            break

        except ValueError:
            print("Error: Invalid number")
            exit()

average = sum(numbers) / n

print("The average is:", round(average, 2))
Logic
  • The program first reads the number of integers.
  • Exception handling validates numeric input.
  • The loop repeatedly accepts integer values.
  • If invalid data is entered, ValueError occurs.
  • The average is calculated after all valid inputs are entered.
Explanation
  • try block contains risky statements.
  • ValueError handles invalid numeric input.
  • sum(numbers) calculates total sum.
  • round() limits output to two decimal places.
Input : 10 20 30 40
Output : The average is: 25.0

Program 2 – Absence of Words / Text Sanitization

Write a program to sanitize text by removing special characters and storing the cleaned content into a file.

Program
import re

try:

    text = input("Enter text: ")

    cleaned = re.sub(r'[^A-Za-z0-9 ]', '', text)

    filename = "clean_text.txt"

    with open(filename, "w") as file:
        file.write(cleaned)

    print("Text saved successfully")

except Exception as e:
    print("Error:", e)
Logic
  • User enters text containing special characters.
  • Regular expression removes unwanted symbols.
  • The cleaned text is stored in a file.
  • Exceptions are handled safely.
Explanation
  • re.sub() replaces unwanted characters.
  • open() creates file in write mode.
  • with automatically closes the file.
Input : Hello@123!!
Output : Text saved successfully

Program 4 – Character Frequency Counter

Write a program to count frequency of characters in a string.

Program
try:

    text = input("Enter String: ")

    frequency = {}

    for ch in text:

        if ch != ' ':

            frequency[ch] = frequency.get(ch, 0) + 1

    print("Character Frequencies:")

    for key, value in frequency.items():
        print(key, ":", value)

except Exception as e:
    print("Error:", e)
Logic
  • User enters a string.
  • Dictionary stores frequency count.
  • Spaces are ignored.
  • Each occurrence increases count value.
Explanation
  • get(ch,0) initializes default count.
  • items() iterates through dictionary.
  • Exception block handles runtime issues.
Input : hello
Output : h=1 e=1 l=2 o=1

Program 5 – User Defined Exception

Write a program to validate register number and mobile number using user-defined exceptions.

Program
class InvalidRegister(Exception):
    pass

class InvalidMobile(Exception):
    pass

try:

    reg = input("Enter Register Number: ")

    if len(reg) != 10:
        raise InvalidRegister("Invalid Register Number")

    mobile = input("Enter Mobile Number: ")

    if len(mobile) != 10:
        raise InvalidMobile("Invalid Mobile Number")

    print("Valid Details")

except InvalidRegister as e:
    print(e)

except InvalidMobile as e:
    print(e)
Logic
  • Two custom exceptions are created.
  • Register number and mobile number are validated.
  • If conditions fail, exceptions are raised.
  • Specific error messages are displayed.
Explanation
  • Custom exception classes inherit Exception class.
  • raise generates user-defined exception.
  • Separate except blocks improve readability.
Input : 98765
Output : Invalid Mobile Number

Program 10 – File Handling Palindrome Checker

Program
try:

    word = input("Enter Word: ")

    with open("word.txt", "w") as file:
        file.write(word)

    with open("word.txt", "r") as file:
        text = file.read()

    cleaned = text.lower()

    if cleaned == cleaned[::-1]:
        print("Palindrome")

    else:
        print("Not Palindrome")

except Exception as e:
    print(e)
Logic
  • User enters a word.
  • The word is stored in a file.
  • The file content is read and checked.
  • Reverse comparison identifies palindrome.
Input : madam
Output : Palindrome
