Overview of Client, Server, and Mobile Application Architecture

Introduction to Web Application Architecture

Web applications follow the Client–Server architecture.

  • Client (Front end): Sends requests (browser / mobile app)
  • Server (Back end): Processes requests and returns responses
  • Communication: HTTP / HTTPS

Introduction to Mobile Application Architecture

Mobile Application Architecture refers to the structured design and organization of a mobile application, defining how components like UI, business logic, and data interact with each other.

Android Architecture Components (Overview)

Used mainly in MVVM (Model–View–ViewModel):

  • ViewModel: Stores UI data
  • LiveData: Observable data holder
  • Room: Database layer
  • Repository: Manages data sources
  • WorkManager: Background tasks

MVVM Components

  • Model: Data
  • View: UI
  • ViewModel: Manages UI-related data

Tool for Mobile Application Development - Android Studio

Android Studio is the official IDE for Android development. It supports modern architectures like MVVM using built-in Android Architecture Components.

  • Recommended Architecture: MVVM (Model–View–ViewModel)
  • Limitations: Limited applications in free trial version and requires higher memory for efficient running.
