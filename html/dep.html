GitHub-Based Online Lab Record System

Department-Level Implementation Guidelines

Objective

To implement a structured, paperless, and industry-oriented online lab record system using GitHub for programming and web-based laboratory courses.

Why GitHub for Lab Records?

  • Completely paperless and eco-friendly
  • Improves studentsâ€™ coding and version control skills
  • Easy plagiarism detection using commit history
  • Supports continuous assessment and transparency
  • NAAC / NBA friendly digital documentation
  • Creates a professional portfolio for students
Faculty Setup (One-Time Process)
  1. Create a central Website for the lab subject
  2. Upload lab manual and experiment folders
  3. Add weekly topics and programme lists
  4. Add Google Form for observation record upload
  5. Ensure weekly student participation
  6. Evaluate based on uploaded records
Student Workflow
  1. Access the lab website
  2. Review current week topics
  3. Download lab manual and complete programs
  4. Upload Word document via Google Form
Evaluation & Assessment
Continuous Assessment
Criteria Description
Correctness Accuracy of output and logic
Code Quality Readability and structure
Explanation Clarity of written description
Commit Discipline Originality and version history
Timely Submission Weekly completion as scheduled

Outcome: A modern, transparent, and scalable lab record system aligned with industry practices and academic quality standards.
