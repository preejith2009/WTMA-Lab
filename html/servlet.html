Server-Side Programming using Java

Understanding Java Servlets, JSP and Session Handling

1️⃣ Introduction to Server-Side Programming

What is Server-Side Programming?

Server-side programming refers to writing code that runs on the web server instead of the client’s browser.

When a user submits a form:

  1. Request goes to server
  2. Server processes logic
  3. Database interaction happens
  4. Response is sent back to client
Client Side vs Server Side
Feature Client-Side Server-Side
Runs On Browser Web Server
Languages HTML, CSS, JavaScript Java, PHP, Python, Node.js
Security Less secure More secure
DB Access No Yes

4️⃣ Java Servlets

What is a Servlet?

A Servlet is a Java class that handles HTTP requests and responses. It runs inside a Servlet Container like Apache Tomcat.

Servlet Lifecycle

  1. init()
  2. service()
  3. destroy()

Simple Servlet Example:

import java.io.*;
import javax.servlet.*;
import javax.servlet.http.*;

public class HelloServlet extends HttpServlet {

    public void doGet(HttpServletRequest request,
                      HttpServletResponse response)
                      throws ServletException, IOException {

        response.setContentType("text/html");
        PrintWriter out = response.getWriter();
        out.println("<h2>Hello WTMA Students!</h2>");
    }
}
5️⃣ JSP (Java Server Pages)

What is JSP?

JSP allows embedding Java code inside HTML.

<html>
<body>
<h2>Welcome to WTMA Portal</h2>
<%
   out.println("Today is: " + new java.util.Date());
%>
</body>
</html>
